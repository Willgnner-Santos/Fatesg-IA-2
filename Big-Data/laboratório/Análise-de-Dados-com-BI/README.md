# Análise de Dados de Vendas com Power BI

Este projeto documenta o processo de análise de dados de vendas de veículos utilizando o **Microsoft Power BI**. O objetivo foi transformar um conjunto de dados brutos em um painel interativo e visualmente atraente, capaz de fornecer insights de negócios de forma clara. O trabalho me permitiu aprofundar meu conhecimento em visualização de dados, storytelling e a criação de dashboards dinâmicos.

## Metodologia e Execução

O trabalho foi conduzido inteiramente no ambiente do Power BI Desktop, seguindo um fluxo de trabalho estruturado, desde a preparação dos dados até a criação de um dashboard completo e interativo.

---

### 1. Ingestão e Preparação dos Dados

A primeira etapa crucial foi a importação do conjunto de dados. Embora a imagem não especifique a fonte, um conjunto de dados sobre vendas de carros foi carregado no Power BI. A etapa de preparação incluiu a limpeza e a transformação dos dados, garantindo que as colunas estivessem no formato e tipo de dados corretos para a análise.

---

### 2. Análise e Visualização

A etapa mais importante foi a criação das visualizações para o dashboard. O painel foi projetado para responder a perguntas-chave sobre as vendas, utilizando uma variedade de gráficos e elementos visuais.

-   **Qual foi o modelo mais vendido em cada ano?**: Um gráfico de barras horizontais detalha a soma do volume de vendas para cada modelo de carro (por exemplo, 7 Series, i8, X1, etc.). Esse gráfico permite uma comparação direta entre os modelos e suas performances de vendas.

-   **Qual o tipo de combustível teve mais vendas ao longo do ano?**: Um gráfico de linhas complexo, com múltiplas linhas para cada tipo de combustível (Diesel, Electric, Hybrid, Petrol), mostra a evolução do volume de vendas ao longo dos anos, de 2010 a 2020. Essa visualização é ideal para identificar tendências e variações de vendas ao longo do tempo.

-   **Qual é o preço médio dos carros vendidos por modelo?**: Um gráfico de barras horizontais mostra a média do preço em USD para cada modelo. Isso permite entender o posicionamento de preço de cada modelo no mercado.

-   **Análise da Transmissão**: Um gráfico de pizza (`donut chart`) divide as vendas por tipo de transmissão (Automática vs. Manual). As etiquetas de dados mostram o volume de vendas em milhões e a porcentagem correspondente, fornecendo um resumo rápido da distribuição de vendas por tipo de transmissão.

-   **Relação entre Quilometragem e Preço**: Um gráfico de dispersão (`scatter plot`) explora a relação entre a soma da quilometragem (KM) e a média do preço em USD. Essa visualização pode revelar padrões ou correlações, como se carros com maior quilometragem tendem a ter preços mais baixos.

-   **Filtros de Anos**: Uma série de cartões separados por ano (2010 a 2018) funcionam como `slicers` (filtros), permitindo que o usuário interaja com o dashboard e visualize os dados para um ano específico, tornando a análise mais dinâmica e interativa.

### Conclusão

Este exercício me proporcionou uma experiência completa e prática na criação de um dashboard de **Business Intelligence**. O projeto reforçou a importância de escolher a visualização correta para a pergunta certa, de forma a contar uma história clara e concisa com os dados. A habilidade de transformar dados brutos em insights acionáveis é fundamental, e este projeto serviu como uma excelente demonstração desse processo.
