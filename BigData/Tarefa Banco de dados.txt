1. Definição e Características

a) Um banco de dados não-relacional (NoSQL) armazena dados sem o uso fixo de tabelas e relações como no SQL. Ele usa formatos como documentos, pares chave-valor, colunas ou grafos.

b) Vantagens:

Alta escalabilidade (lida bem com grandes volumes de dados).

Flexibilidade de estrutura (não exige schema fixo).

Desempenho superior em consultas específicas e distribuídas.

Facilita o armazenamento de dados complexos (JSON, objetos etc.).

c) Situações práticas:

Redes sociais (dados dinâmicos e conexões entre usuários).

Aplicativos de e-commerce (catálogo de produtos variado).

Jogos online (dados em tempo real e alta demanda de leitura).

2. Tipos de Bancos de Dados Não-Relacionais

a) Diferenças principais:

Orientados a Documentos: armazenam dados como documentos (JSON, BSON). → Ex.: MongoDB.

Chave-Valor: cada registro é um par chave + valor. → Ex.: Redis.

Colunas Amplas: organizam dados em colunas em vez de linhas. → Ex.: Cassandra.

Grafos: focam em nós e relações entre eles. → Ex.: Neo4j.

b) Exemplos:

Tipo	Ferramenta popular
Documentos	MongoDB, CouchDB
Chave-Valor	Redis, DynamoDB
Colunas Amplas	Apache Cassandra, HBase
Grafos	Neo4j, ArangoDB
3. Comparação Prática

a) A principal diferença é que o modelo relacional usa tabelas com chaves e joins, enquanto o modelo de grafos representa relações diretas entre entidades (nós e arestas).

b) Para redes sociais, o mais adequado é o banco de grafos (como Neo4j), pois ele lida melhor com conexões entre usuários, amizades e recomendações.

4. Escalabilidade e Flexibilidade

a) Escalabilidade horizontal = adicionar mais servidores para dividir a carga, em vez de aumentar o poder de um só servidor. Isso é fácil no NoSQL, pois os dados podem ser distribuídos entre várias máquinas.

b) A flexibilidade de schema permite adicionar novos campos aos documentos sem precisar alterar toda a estrutura, ideal para dados que mudam com frequência.

5. Estudo de Caso

a) Netflix usa o Cassandra (colunas amplas).

b) Ele é usado para armazenar informações sobre usuários, histórico de reprodução e recomendações.
Benefícios:

Alta disponibilidade global.

Escalabilidade automática.

Rapidez no acesso aos dados, mesmo com milhões de usuários conectados simultaneamente.
